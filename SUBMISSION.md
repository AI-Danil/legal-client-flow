## Что сделано

Собрал рабочий прототип мини-CRM для юриста: таблица клиентов, добавление клиента с именем/телефоном/статусом, переключение статуса по цепочке `Новый -> В работе -> Закрыт`, счетчики по каждому статусу. Данные сохраняются в `localStorage`, поэтому прототип работает без backend и не теряет клиентов после обновления страницы.

Бонус: добавил webhook-настройку. Если указать URL из Make, n8n, Zapier или своего backend, при добавлении клиента прототип отправляет `POST` с данными клиента, и из этого можно собрать Telegram/email-уведомление.

## Лог

Сам сделал продуктовый сценарий, структуру интерфейса, модель данных, логику статусов, локальное сохранение, проверку сборки и функциональный тест. AI использовал как pair-programmer для ускорения React/TypeScript/CSS-реализации, генерации boilerplate и быстрой проверки UX-формулировок.

Выбрал React + Vite + TypeScript, потому что это самый быстрый путь собрать живой фронтенд-прототип за 2-3 часа и затем без переделки опубликовать его на Vercel/Netlify. Backend не стал добавлять на MVP-этапе: для тестового сценария достаточно `localStorage`, а при развитии данные можно заменить на Supabase/Firebase.
